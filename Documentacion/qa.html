Flujos de QA Manual y Datos de Prueba

Este documento detalla los flujos de prueba manual y proporciona datos para realizar QA exhaustivo del sistema de gestión de archivos Excel. A continuación se presentan escenarios de prueba detallados con datos específicos para verificar el correcto funcionamiento del sistema.

1. Pruebas de Autenticación y Autorización COMPLETADO

1.1. Inicio de Sesión

  1. Acceder a la página de inicio de sesión
  2. Introducir credenciales válidas
  3. Verificar redirección al dashboard
  4. Verificar que el menú muestra opciones adecuadas al rol
  5. Cerrar sesión y confirmar redirección a página de login

1.2. Verificación de Control de Acceso

  1. Iniciar sesión con usuario responsable
  2. Intentar acceder a rutas administrativas mediante URL directa
  3. Verificar redirección o mensaje de error
  4. Intentar acceder a archivos/carpetas fuera de su área

  4. Completar formulario con datos válidos
  5. Asignar rol y áreas/subáreas
  6. Guardar y verificar creación exitosa
  7. Verificar email de notificación (si aplica)

3. Pruebas de Gestión de Archivos PENDIENTE

3.1. Carga de Archivos

  1. Iniciar sesión como usuario responsable
  2. Navegar a un área/carpeta con permisos de escritura
  3. Hacer clic en "Subir Archivo"
  4. Seleccionar un archivo Excel válido
  5. Completar metadatos (si aplica)
  6. Confirmar carga
  7. Verificar que el archivo aparece en la lista

Archivos de Prueba:

  • Archivo Pequeño: reporte_mensual.xlsx (50KB) - Debe cargarse rápidamente
  • Archivo Mediano: transacciones_trimestre.xlsx (5MB) - Debe mostrar barra de progreso
  • Archivo Grande: datos_anuales.xlsx (20MB) - Debe manejar carga prolongada
  • Formato Inválido: documento.docx - Debe rechazar el archivo
  • Excel Corrupto: archivo_dañado.xlsx - Debe detectar y rechazar

Instrucciones detalladas para pruebas de carga:

  1. Iniciar sesión como usuario@acme.com (Usuario2023!)
  2. Navegar a "Explorador de Archivos" > Área asignada
  3. Usar los siguientes métodos de carga para probar:
    • Botón "Subir Archivo"
    • Arrastrar y soltar (drag & drop) directamente en la interfaz
    • Botón "Subir Múltiples Archivos" (si existe)
  4. Para cada archivo de prueba, realizar los siguientes pasos:
    • Seleccionar el archivo correspondiente
    • Completar formulario de metadatos (Título, Descripción, Etiquetas)
    • Verificar la barra de progreso durante la carga
    • Revisar el mensaje de confirmación al finalizar
    • Comprobar que el archivo aparece correctamente en la lista
    • Verificar que los metadatos se guardaron correctamente
  5. Para archivos no válidos (docx, corrupto), verificar:
    • Mensaje de error claro y específico
    • No se permite completar la carga
    • Sistema se recupera adecuadamente después del error

Prueba específica para archivos grandes (datos_anuales.xlsx):

  1. Cronometrar el tiempo de carga
  2. Verificar que la barra de progreso funciona correctamente
  3. Intentar navegar a otra sección durante la carga para comprobar si:
    • La carga continúa en segundo plano
    • Se muestra advertencia antes de abandonar la página
  4. Verificar notificación al completarse la carga

  2. Navegar al explorador de archivos
  3. Crear estructura de carpetas anidadas (3 niveles)
  4. Mover archivos entre carpetas con drag & drop
  5. Verificar que los permisos se mantienen al mover
  6. Verificar navegación a través de breadcrumbs

Instrucciones detalladas para crear y organizar carpetas:

  2. Navegar a "Explorador de Archivos" > "Operaciones"
  3. Crear carpeta raíz "Proyectos 2023":
    • Clic en botón "Nueva Carpeta"
    • Introducir nombre "Proyectos 2023"
    • Opcional: Añadir descripción
    • Confirmar creación
  4. Entrar en la carpeta "Proyectos 2023" y crear subcarpetas:
    • Crear carpetas "Q1", "Q2" y "Documentos Compartidos" siguiendo el mismo proceso
  5. Entrar en la carpeta "Q1" y crear las subcarpetas "Enero", "Febrero" y "Marzo"
  6. Entrar en la carpeta "Q2" y crear las subcarpetas "Abril", "Mayo" y "Junio"
  7. Verificar la estructura creada navegando entre carpetas
  8. Subir al menos un archivo a cada carpeta del último nivel
  9. Probar organización:
    • Seleccionar archivo de "Enero" y arrastrarlo a "Febrero"
    • Mover un archivo desde "Q1" a "Q2" usando el menú contextual
    • Seleccionar múltiples archivos y moverlos a "Documentos Compartidos"
  10. Verificar la navegación usando breadcrumbs (miga de pan) en la parte superior
  11. Comprobar que la estructura de carpetas se mantiene al cerrar sesión y volver a ingresar

Prueba de permisos en movimiento de archivos:

  1. Iniciar sesión como admin@acme.com

Usar "presupuesto_departamental.xlsx" y crear 3 versiones con cambios progresivos:

  • V2: Modificar cifras en hoja "Gastos"
  • V3: Añadir nueva hoja "Proyecciones"

Instrucciones detalladas para pruebas de versionado:

  1. Ubicación del archivo: "ACME Corporation/Finanzas/Presupuestos/presupuesto_departamental.xlsx"
  2. Para generar la versión 1 (original):
    • Iniciar sesión como finanzas@acme.com (Finanzas2023!)
    • Navegar a la ubicación del archivo
    • Verificar que existe o subirlo si es necesario
    • Marcar esta versión como "Línea base"
  3. Para generar la versión 2:
    • Descargar el archivo actual
    • Abrir en Excel y modificar valores en la hoja "Gastos"
    • Guardar localmente
    • En la plataforma, seleccionar el archivo y hacer clic en "Subir nueva versión"
    • Seleccionar el archivo modificado
    • Añadir comentario: "Actualización de cifras de gastos trimestrales"
    • Confirmar actualización
  4. Para generar la versión 3:
    • Descargar la versión 2
    • Abrir en Excel
    • Añadir nueva hoja llamada "Proyecciones"
    • Completar con datos de proyección financiera
    • Guardar localmente
    • Subir como nueva versión con comentario: "Añadida hoja de proyecciones financieras"
  5. Pruebas de visualización de versiones:
    • En la vista de detalles del archivo, localizar pestaña o sección "Historial"
    • Verificar que se muestran las 3 versiones con fechas y usuarios
    • Comprobar que se muestran los comentarios de cada versión
  6. Pruebas de comparación de versiones (si la función está disponible):
    • Seleccionar versiones 1 y 3
    • Usar función "Comparar"
    • Verificar que se destacan los cambios (nueva hoja, valores modificados)
  7. Pruebas de reversión:
    • Seleccionar la versión 1
    • Usar función "Restaurar esta versión"
    • Confirmar la acción
    • Verificar que el archivo ahora muestra el contenido de la versión 1
    • Comprobar que se crea una nueva entrada en el historial (versión 4) que indica la restauración

Prueba de permisos de versionado:

  1. Iniciar sesión como usuario con permisos limitados
  2. Intentar ver el historial de versiones (debe ser posible)
  3. Intentar descargar versiones anteriores (debe ser posible)
  4. Intentar revertir a una versión anterior (debe ser denegado)
  5. Intentar eliminar versiones antiguas (debe ser denegado)

5. Pruebas de Interfaz y Experiencia de Usuario PENDIENTE

5.1. Validación de Formularios

  1. Probar formularios de creación/edición con datos inválidos
  2. Verificar mensajes de error claros
  3. Comprobar validación en tiempo real (si aplica)
  4. Verificar conservación de datos válidos al corregir errores

Escenarios de Prueba:

  • Email inválido: "usuario@incompleto"
  • Contraseña débil: "123456"
  • Campos obligatorios vacíos
  • Valores duplicados (email ya existente)
  • Valores fuera de rango (números negativos en campos positivos)

Instrucciones detalladas para pruebas de validación de formularios:

Formulario de Registro de Usuario:
  1. Iniciar sesión como admin@acme.com
  2. Navegar a "Configuración" > "Gestión de Usuarios" > "Añadir Usuario"
  3. Realizar las siguientes pruebas una por una:
    • Email inválido: Introducir "usuario@incompleto" y verificar que se muestra error específico
    • Email duplicado: Intentar crear usuario con email "control@acme.com" (ya existente) y verificar mensaje de error
    • Contraseña débil: Probar con "123456" y verificar que se rechaza con mensaje explicativo
    • Campos obligatorios: Dejar nombre vacío y verificar que no permite continuar
    • Selección de rol: Intentar guardar sin seleccionar rol y verificar error
Formulario de Creación de Carpeta:
  1. Navegar al explorador de archivos
  2. Iniciar creación de carpeta
  3. Realizar las siguientes pruebas:
    • Nombre vacío: Intentar crear carpeta sin nombre
    • Caracteres especiales: Probar nombres con caracteres como "*/\\:?\"<>|"
    • Nombre duplicado: Intentar crear carpeta con nombre ya existente
Formulario de Carga de Archivo:
  1. Iniciar proceso de carga de archivo
  2. Probar validación de:
    • Archivo sin seleccionar: Intentar continuar sin elegir archivo
    • Metadatos incompletos: Omitir campos obligatorios
    • Descripción muy larga: Introducir texto excesivamente largo
  3. Verificar que tras corregir errores, los campos válidos mantienen su información

Herramientas para pruebas:
